Experiment 1: HTML5 Form

About the experiment

HTML 5 provides various new form elements which are frequently used these days.
Support for these kind of elements will help to simplify the web development in a way that
the developer can concentrate on the program logic rather than the representation and validation of the input data.
Following are some of the new form elements:

  • Date Picker
  • Email
  • Datalist
  • Number

Date Picker:

The <input type="date"> is used for input fields that should contain a date. Depending on browser support, a date picker can show up in the input field.

Email:

The <input type="email"> is used for input fields that should contain an e-mail address. Depending on browser support, the e-mail address can be automatically validated when submitted. Some smartphones recognize the email type, and adds ".com" to the keyboard to match email input.

Number:

The <input type="number"> is used for input fields that should contain a numeric value. You can set restrictions on the numbers. Depending on browser support, the restrictions can apply to the input field.

Datalist:

The <datalist> element specifies a list of pre-defined options for an <input> element. Users will see a drop-down list of pre-defined options as they input data.
