Experiment 23 : Angular JS Routing

About the experiment

This page gives the look and feel as if we are on the same page where as actually this page is navigating it through different pages. The angularJS routing property is loading a new page without refreshing the whole page. This specific property of angularJS makes it stand unique among other similar frameworks. 'ngRoute' is a service that is used for naviagtion of this sort. The configuration of ngRoute takes in the actual path and maps it to the corresponding html file and controller. The new file so mapped will use the mapped the attributes and methods in its mapped controller. Every page will have its won controller.


Code Snippet:

app.config(function ($routeProvider) {
$routeProvider
.when('/home', {
templateUrl: 'WebsitePages/home.html'
})

.when('/login', {
templateUrl: 'WebsitePages/Login.html'
})

.when('/search', {
templateUrl: 'WebsitePages/search.html',
controller: 'searchController'
})

.when('/favorites', {
templateUrl: 'WebsitePages/NoFavorites.html'
});

});

When user clicks a link, it generates a path which is then searched in the config for the url match. Once the match is found, the corresponsing HTML file is loaded. The favorites that the user selects will not be avaiable as the data is not carried out to the next page using service. Its covered in the next experiment.
